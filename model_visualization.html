Conv-TasNet + WavLM  ·  dwconvFuse

Conv_TasNet_wavlm_dwconvFuse.py — 2-speaker separation with frozen-WavLM semantic features injected after the depthwise conv of every TCN block via gated cross-attention

1 · Concept

The core idea: a time-domain separator (Conv-TasNet) hears how the mixture sounds, while a frozen WavLM encoder understands what is being said. Cross-attention lets every acoustic frame query the semantic sequence, and a learnable gate (initialized nearly closed, σ(−2) ≈ 0.12) blends the semantic hint in only as far as it actually helps separation.

acoustic (trainable) semantic (frozen WavLM) fusion separated outputs

2 · End-to-end pipeline

Data flow of ConvTasNet.forward(): the raw mixture feeds both branches; the separation module fuses them and produces per-speaker masks that are applied to the encoder output w before decoding. Training uses permutation-invariant SI-SNR (SI_SNR.si_snr_loss).

3 · Architecture detail — TCN block with dwconvFuse injection

Left: the separation stack — R=3 repeats, each owning one shared CrossAttnDelta used by all of its X=8 dilated blocks (dilation 2⁰…2⁷). Right: inside one Conv1D_Block — the semantic delta is computed from the post-depthwise-conv features (hence “dwconvFuse”) and merged through GatedResidual before the 1×1 skip projection.

convolutional (audio) attention / fusion WavLM semantic input

Hyper-parameters (defaults in ConvTasNet.__init__): N=512, L=16, B=128, H=512, P=3, X=8, R=3, attn_dim=128, dropout=0.1, gLN, non-causal, num_spks=2, freeze_wavlm=True. The SemanticDownsampler exists in the file but is commented out in SeparationModule.forward — attention currently runs over the full WavLM sequence.
