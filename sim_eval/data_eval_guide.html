0. 环境准备

核心库（torch / torchaudio / numpy / scipy / soundfile / librosa / transformers）一般已经装好。 评估额外依赖以下三个指标库 + 一个 ASR 模型（用于算 WER）：

# 指标库
pip install pesq pystoi jiwer

# WER 用的 ASR：脚本默认用 transformers 自动下载 openai/whisper-tiny.en（约 40MB）
# 首次运行会联网下载，之后会走本地缓存
提示： 如果机器不能联网，加 --no-wer 跳过 WER（其余四个指标照常计算）。 本仓库新版 torchaudio.load 需要 torchcodec，所以 evaluate.py 内部统一改用 soundfile 读音频，无需额外安装。

1. 整体流程概览

三个脚本各司其职，全部通过 run_inference.sh 调用：

脚本作用对应子命令
synthesize_datasets.py仿真 / 合成两说话人混合数据（四种条件）synth
Separation_nets.py（仓库已有）用训练好的模型做分离推理，输出 spk1/spk2run
evaluate.py对分离结果打分（5 个指标 + PIT 排列）eval

典型流水线：仿真数据 → 模型分离 → 评估打分。

仿真混合音  ──►  模型推理        ──►  评估
synth            run (Separation_nets.py)   eval
                 输出 spk1/ spk2/           输出指标表格

2. 数据仿真

2.1 一条命令生成

# 每种条件生成 3 条混合音，输出到 synth_data/
./run_inference.sh synth 3 synth_data

# 等价于直接调用 python：
python synthesize_datasets.py --n-pairs 3 --out synth_data

2.2 四种声学条件（2×2 鲁棒性网格）

条件含义混合公式
clean纯净s1 + s2
noise加噪s1 + s2 + 噪声（SNR 0–10 dB 随机）
reverb混响reverb(s1) + reverb(s2)
noise_reverb加噪 + 混响reverb(s1) + reverb(s2) + 噪声

2.3 输出目录结构

每个条件一个子目录，布局与 LibriMix wav8k/max/<set>/ 完全一致，单声道、8000 Hz：

synth_data/
  ├─ clean/
  │    ├─ mix/      ← 喂给分离模型的混合音  .wav
  │    ├─ s1/       ← 干净参考源 1（评估用）
  │    ├─ s2/       ← 干净参考源 2
  │    ├─ text_spk1 ← " 转写文本" （算 WER 用）
  │    ├─ text_spk2
  │    └─ metadata.csv
  ├─ noise/  ...（同上结构）
  ├─ reverb/
  └─ noise_reverb/

2.4 用真实语音作为声源（推荐）

默认用「类语音合成信号」，能让脚本在任何机器上跑通（CI / 无数据冒烟测试）。 若想用真实语音（如 LibriSpeech 切片），把一个装满 .wav 的目录传给 --source-dir，脚本会两两配对：

python synthesize_datasets.py \
    --n-pairs 50 \
    --source-dir /path/to/librispeech_wavs \
    --out synth_data_real

2.5 常用参数

参数默认说明
--n-pairs3每种条件生成多少条混合音
--outsynth_data输出根目录
--dur3.0合成信号时长（秒），仅合成模式生效
--source-dir（空）真实语音目录；不填则用合成信号
--conditions四种全做只做子集，如 --conditions clean noise
--seed0随机种子，保证可复现
预期输出（synth 子命令）： 
Per-condition mixture counts:
----------------------------------------
  clean             3 mixtures
  noise             3 mixtures
  reverb            3 mixtures
  noise_reverb      3 mixtures
----------------------------------------
  TOTAL            12 mixtures

3. 结果评估

3.1 模型的真实输出格式（评估的输入）

分离模型（Separation_nets.py）把结果写成下面的布局——这正是 evaluate.py 读取的格式：

<save_path>/
  ├─ spk1/.wav   ← 估计的说话人 1
  └─ spk2/.wav   ← 估计的说话人 2

· 固定 2 个源，目录名就叫 spk1 / spk2
·  与混合音、参考源 s1/s2 的文件名完全相同（这是对齐的关键）
· 单声道，8000 Hz

3.2 评估命令

# 最简：自动寻找输出目录旁边的 _refs/ 参考（冒烟测试时）
./run_inference.sh eval <输出目录>

# 真实场景：显式指定参考源 / 混合音 / 转写文本
python evaluate.py enhanced/myrun \
    --ref-dir   /lustre/.../LibriMix/Libri2Mix/wav8k/max/test \
    --mix-dir   /lustre/.../LibriMix/Libri2Mix/wav8k/max/test/mix_both \
    --text-spk1 data/text/test/text_spk1 \
    --text-spk2 data/text/test/text_spk2

3.3 五个评估指标

指标含义方向
SI-SDR尺度不变信号失真比（dB），衡量分离质量越大越好 ↑
SI-SDRi相对「混合音直接当估计」基线的 SI-SDR 提升量越大越好 ↑
PESQ感知语音质量（窄带 nb @ 8 kHz）越大越好 ↑（1–4.5）
STOI短时客观可懂度（0–1）越大越好 ↑
WER分离音经 ASR 转写后与参考文本的词错误率越小越好 ↓
关于 PIT 排列： 两个估计源到底对应哪个参考源是未知的。评估时按 SI-SDR 最大化 在两种排列里自动选最优配对（输出里的 perm=01 / perm=10 就是选中的排列）， 其余指标都在选中的配对下计算。这套排列逻辑与指标公式是标准实现，请勿改动。

3.5 常用参数

参数默认说明
--ref-dir自动找 _refs含 s1/ s2/ 的参考源目录
--mix-dir（空）混合音目录；提供后才能算 SI-SDRi
--text-spk1/2（空）参考转写（text_spkN 或 GT csv）；提供后才算 WER
--limit0（全部）只评估前 N 条，调试用
--no-wer—跳过 ASR/WER（不联网时用）
--asr-modelwhisper-tiny.en换更大的 ASR，如 openai/whisper-base.en
--sr8000采样率，与模型输出一致

4. 一键「推理 + 评估」

run_eval 子命令把模型分离和评估串成一步：先调用 Separation_nets.py 生成 spk1/spk2，再自动评估。

# 用环境变量指定 checkpoint / 网络结构 / 评估集，再一键跑
MODEL_CKPT=/path/best.pt \
MODEL_NET=Net_dwAtt_film \
EVAL_SET=test \
LIBRIMIX_ROOT=/lustre/.../LibriMix/Libri2Mix/wav8k/max \
./run_inference.sh run_eval  tt_mix.scp  enhanced/myrun  --limit 50

参考源 / 混合音 / 转写默认按 LIBRIMIX_ROOT + EVAL_SET 推导（与 inferece.sh、SI_SNR_eval.py 一致），存在才传入；也可在命令末尾追加 evaluate.py 的任意参数（如 --no-wer）。

注意： run / run_eval 的推理那一步需要训练好的 checkpoint、GPU、WavLM 权重以及 LibriMix 数据； 没有这些时请先用 synth 造数据、或直接用已有的 spk1/spk2 目录跑 eval。

5. 常见问题

Q1：合成数据上 WER 很大（远大于 1）正常吗？

正常。合成的「类语音」信号不是真实语音，Whisper 会瞎猜导致大量插入错误， WER =（替换+删除+插入）/ 参考词数，插入太多就会 > 1。这是真实计算结果，不是 bug。 想看有意义的 WER，请用 --source-dir 喂真实语音。

Q2：只想快速验证脚本能跑通，但没有模型？

用 fixture 造一份「模型格式」的假输出（参考源 + 噪声、部分打乱顺序以检验 PIT），再评估：

./run_inference.sh fixture fixture_out 5
./run_inference.sh eval    fixture_out

Q3：评估真实 LibriMix 结果时参考源放哪？

用 --ref-dir 指向 .../wav8k/max/<test|dev>（里面有 s1/ s2/）， --mix-dir 指向同级 mix_both，转写用本仓库 data/text/<set>/text_spk1。

Q4：想改输出目录命名（不叫 spk1/spk2）？

只改 evaluate.py 顶部 ADAPT-THIS 区块的 EST_SUBDIRS / REF_SUBDIRS 和四个路径钩子函数即可，指标与 PIT 逻辑无需动。
